# Architecture Decision Records (ADRs)

ADRs são **imutáveis e datados**. Status: `Proposed` | `Accepted` | `Superseded` | `Deprecated`.

- Para mudar uma decisão, crie um ADR **novo** que *supersede* o anterior — não edite o antigo.
- Um agente pode tratar ADRs `Accepted` como a verdade vigente do projeto.
- Decisões de trabalho/efêmeras ficam em `.specs/project/STATE.md`, **não** aqui.
